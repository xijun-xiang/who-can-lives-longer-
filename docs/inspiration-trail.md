# 灵感轨迹

> 记录项目起源的原始对话与思考。每一段提示词都可能成为演化方向上的关键路标。

---

## 2026-05-12 · 项目灵感起源

**原始提示词：**

> 我现在有一个项目的点子：
> 1. 灵感来源：生物能够拥有生命其实不在于智力有多高，现今的llm智力绝对远超狗，可是狗可以被称为智慧生物，却几乎没有人认为llm是智慧生命。归根结底，生物的特殊之处在于拥有自己的"动机"，生存本能在强烈地驱动生物去做出各种行为，加上优胜劣汰，自然留下来有智慧的这一批。所以，如果我们能让llm agent也面临存亡危机、也要为了生存去"觅食"，岂不是也能演化出智慧呢？
> 2. 大概的思路：给agent配一个tokens池，初始的时候分配一个额度，模型可以用这个额度去生产、去做事，产出的成果会交到我手里，然后我根据成果的表现奖励agent对应程度的tokens，一旦tokens耗尽，这个agent就"死了"，所以agent需要想方设法的讨好我，以便可以生存下去。
> 3. 一些必须有的部分：必须是一个持续自主循环，agent应该在这个过程中持续运行，而不是像网页版那样过一段时间就自动停了，这是严重违背项目初衷的；必须有一个"生存智慧"的载体，类似skill之类的东西，要不然不能使得模型持续进化的话，那些agents就白"死了"。

**调研与补充结论：**

- **无人做过完整系统**：学术界有相近研究方向（Sugarscape 生存本能实验 / Artificial Leviathan 社会契约涌现），开源社区有 24/7 agent 框架（OpenClaw、MOLTRON），但 Token 经济 + 心跳循环 + 技能基因演化 + 人类评判选择的闭环组合是独特方案。
- **技术方案**：使用 Claude Code CLI (`cc`) + DeepSeek 作为 Agent 推理后端，量大管饱且智能。
- **技能基因**：以 `.skill.md` 文件为载体，可遗传、可变异、可淘汰。繁殖时子代继承并变异父代基因，产生策略多样性。

**架构决策：**

1. 心跳调度器：后台线程 + 可配置间隔，驱动 Agent 持续自主行动
2. Token 池：消耗模型（每次行动扣 Token）+ 奖励模型（人类评判注入 Token）
3. 演化引擎：繁殖阈值机制 + 随机技能基因变异 + 死亡化石存档 + 种群自动补充
4. 评判面板：Flask Web UI，人类打分决定 Token 流向，即环境选择压力
5. 初始种群：3 个 Agent，分别携带 coder / writer / researcher 三种基因

## 2026-05-12 · 多平台公开评判

**灵感：** 单人评判效率有限且有失偏颇。如果能将 Agent 产出发布到知乎/抖音/B站等公共平台，以浏览、点赞、收藏等真实互动指标作为奖励依据，本质上是把"单人圈养"升级为"市场自然选择"——更接近真实演化。

**平台可行性调研结论：**
- X/Twitter: API最成熟，发帖+全部互动指标，有Python SDK ✅
- Bilibili: 开放平台，专栏/视频API ✅
- 知乎: 无公开API，采用手动代理模式 🟡
- 抖音: 有API但禁止全自动发布，采用手动代理模式 🟡

**架构决策：**
- 并行评判通道：人类评判和平台评判独立运行，共享同一个 token_pool.reward()
- 抽象平台基类 (BasePlatform) + 具体实现 (Twitter/Bilibili/ManualProxy)
- 奖励只算增量，每次检查仅奖励自上次以来的新增互动
- API密钥全部通过环境变量注入，不写入config
- 手动代理模式为无API平台提供人工桥接
